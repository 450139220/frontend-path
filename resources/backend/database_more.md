### 1. **基础知识：理解数据库和关系型数据库**

- **数据库基础**：了解什么是数据库、什么是关系型数据库（如 MySQL）和非关系型数据库（如 MongoDB）。
- **表、字段、记录**：知道什么是表（Table），字段（Column），记录（Row）。
- **主键与外键**：理解主键（Primary Key）如何唯一标识一条记录，外键（Foreign Key）如何建立表之间的关系。

### 2. **常见的数据库操作**

- **增删改查（CRUD）**：
  - `SELECT`（查询）：如何从表中检索数据。
  - `INSERT`（插入）：如何向表中添加数据。
  - `UPDATE`（更新）：如何修改表中的数据。
  - `DELETE`（删除）：如何删除表中的数据。
- **条件查询**：使用 `WHERE` 进行筛选，理解 `AND`, `OR`, `LIKE`, `BETWEEN` 等常见查询条件。
- **排序与分页**：使用 `ORDER BY` 排序数据，使用 `LIMIT` 实现分页。

### 3. **数据库设计**

- **ER 图设计**：掌握如何从业务需求出发设计 ER 图，确定实体、属性、关系。
- **表结构设计**：从 ER 图到数据库表的转换，合理设计表结构（字段类型、大小等）。
- **规范化**：避免数据冗余，进行第一范式、第二范式等规范化设计。

### 4. **索引与查询优化**

- **索引**：理解索引的作用（加速查询），并掌握在常查询字段上添加索引的方式。
- **查询优化**：理解常见的查询性能问题，学习如何避免全表扫描，使用索引提高查询效率。

### 5. **事务和数据一致性**

- **事务**：了解数据库事务的ACID特性（原子性、一致性、隔离性、持久性），如何使用事务保证数据一致性。
- **锁机制**：理解数据库的行锁和表锁，掌握事务的并发控制。

### 6. **数据库与后端开发结合**

- **与后端交互**：学习如何通过 API（如 RESTful API 或 GraphQL）与数据库进行交互。比如前端发请求给后端，后端处理数据库操作后返回数据。
- **ORM（对象关系映射）**：了解如何通过 ORM（如 Sequelize、TypeORM 等）在 JavaScript/TypeScript 中操作数据库，而不需要直接写 SQL 语句。

### 一、常见数据库操作 SQL 示例

#### 1. **查询操作（SELECT）**

- 查询表中所有数据：

  ```sql
  SELECT * FROM Student;
  ```

- 查询特定字段（只查询姓名和学号）：

  ```sql
  SELECT sid, name FROM Student;
  ```

- 条件查询（查找成绩大于 90 的学生）：

  ```sql
  SELECT * FROM Student WHERE score > 90;
  ```

- 排序查询（按成绩升序排列）：

  ```sql
  SELECT * FROM Student ORDER BY score ASC;
  ```

- 分页查询（每页 10 条数据，查询第 2 页数据）：

  ```sql
  SELECT * FROM Student LIMIT 10 OFFSET 10;
  ```

#### 2. **插入操作（INSERT）**

- 插入一条记录：

  ```sql
  INSERT INTO Student (sid, name, score) VALUES (1, '张三', 85);
  ```

- 插入多条记录：

  ```sql
  INSERT INTO Student (sid, name, score) VALUES 
  (2, '李四', 92),
  (3, '王五', 78);
  ```

#### 3. **更新操作（UPDATE）**

- 更新某一记录的值：

  ```sql
  UPDATE Student SET score = 95 WHERE sid = 1;
  ```

#### 4. **删除操作（DELETE）**

- 删除一条记录：

  ```sql
  DELETE FROM Student WHERE sid = 1;
  ```

- 删除所有记录：

  ```sql
  DELETE FROM Student;
  ```

### 二、数据库设计扩展讲解

#### 1. **如何从业务需求设计数据库（ER 图）**

- **业务需求分析**：假设你需要为一个学校的学生选课系统设计数据库。
- **确定实体**：在这个系统中，主要的实体是：学生（Student）、课程（Course）、选课（Enrollment）。
  - 学生表：保存学生的基本信息。
  - 课程表：保存课程的基本信息。
  - 选课表：保存学生选课的记录。

#### 2. **从 ER 图 到 关系表的转换**

- **Student 表**：记录学生信息

  ```sql
  CREATE TABLE Student (
      sid INT PRIMARY KEY,       -- 学号
      name VARCHAR(50),          -- 姓名
      score DECIMAL(5,2)         -- 成绩
  );
  ```

- **Course 表**：记录课程信息

  ```sql
  CREATE TABLE Course (
      cid INT PRIMARY KEY,       -- 课程号
      title VARCHAR(100)         -- 课程名
  );
  ```

- **Enrollment 表**：记录学生选课信息

  ```sql
  CREATE TABLE Enrollment (
      sid INT,                   -- 学号（外键）
      cid INT,                   -- 课程号（外键）
      score DECIMAL(5,2),        -- 成绩
      PRIMARY KEY (sid, cid),
      FOREIGN KEY (sid) REFERENCES Student(sid),
      FOREIGN KEY (cid) REFERENCES Course(cid)
  );
  ```

#### 3. **范式化设计**

- **第一范式（1NF）**：每个字段保持原子性，即每个字段只能存储单一值。
  - 示例：不要将学生的多个电话号码存储在同一字段里，应该拆分为多行记录。
- **第二范式（2NF）**：每个非主键字段完全依赖于主键。
  - 示例：如果学生表中有“课程”字段，而课程信息在其他表中存在，应该拆分成两个表，避免冗余。
- **第三范式（3NF）**：非主键字段不依赖于其他非主键字段。
  - 示例：如果学生表中同时存有“课程名称”和“课程号”，课程名称应该拆分到课程表中，而不应该和学生信息存储在同一个表。

#### 4. **规范化的目的**

- 主要目的是**减少冗余**，提高数据的一致性和完整性。例如，避免在多个地方存储相同的课程信息，减少更新时的异常（如更新课程名称时只需要在课程表更新）。

#### 5. **外键与表关系**

- 外键（Foreign Key）用于建立表与表之间的关系：
  - **一对多关系**：例如，一个学生可以选多门课程，一门课程也可以有多个学生选择。在 `Enrollment` 表中，通过 `sid` 和 `cid` 分别与 `Student` 和 `Course` 表建立外键关联。

------

### 总结

- **数据库设计**从 ER 图开始，转换成表结构，再进行规范化设计。
- **常见的数据库操作**包括：增（INSERT）、查（SELECT）、改（UPDATE）、删（DELETE）。
- **表之间的关系**（外键）确保数据的完整性和一致性。
